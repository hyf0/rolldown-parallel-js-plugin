# Product Capability Ledger

This executable ledger prevents mechanical or resource speedups from being described as a product crossover. Every Vue and MDX semantic capability is exactly `pass`, `product-failure`, or `not-tested`; the set includes plugin context and deterministic worker-local caches in addition to code, maps, metadata, diagnostics, state, ordering, lifecycle, virtual modules, failure, and shutdown. A family-level `productCrossover` can become true only when every semantic capability passes and both a mechanical and resource-acceptable performance crossover are established. The verifier recomputes every local artifact hash and checks each capability against its exact allowed evidence IDs.

Controlled Vue ledger evidence lives under content address `82ff7743fcd200d2c0df5efb8b740b6be9585b66a164a9072be483956119b8b6`. The bundle retains the unchanged admission and correctness raw reports and pointers, a Git commit/tree/blob proof for the complete ignored-evidence harness scope, and readable copies of the two source files that establish the transform-map and coordinator-owned virtual-module boundaries. The verifier recomputes the bundle address, validates the raw/pointer bindings, walks the exact committed Git tree, matches both source copies to the snapshot blobs, and derives the 5,000-SFC equality claim from the raw runs and pointer golden. It does not rely on a hand-authored source record. This is a repository-local claim and provenance bundle, not an offline replay bundle: the public Vue corpora, package payloads, Node binary, lifecycle-fixed Rolldown distribution, and generated outputs remain bound by hashes and Git pins rather than duplicated here.
